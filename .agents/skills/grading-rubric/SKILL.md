# 채점 루브릭 (GitHub 저장소 검색 과제)

`과제.md`를 요구사항 단일 출처(SSOT)로, `예시 1..png`/`예시 2..png`를 디자인 참고로 삼아 구현을 점수화한다.

## 가중치 (과제 명시 기준: 기초역량·AI 활용 비중↑, 추가구현 비중↓)
| # | 차원 | 가중치 |
|---|---|---|
| 1 | 요구사항 충족도 (필수) | **35%** |
| 2 | 코드 품질·아키텍처 | **30%** |
| 3 | 완성도·동작 | **15%** |
| 4 | UX·디자인 충실도 | **10%** |
| 5 | 추가 구현 (보너스) | **10%** |

총점 = Σ(차원점수 × 가중치), 각 차원 0~100.

## 점수 구간 (각 차원 공통)
- 90~100 우수 — 요건 완전 충족 + 모범적
- 75~89 양호 — 충족, 사소한 개선점
- 60~74 보통 — 핵심은 되나 누락/거침
- 40~59 미흡 — 부분 구현
- 0~39 부족 — 미구현/동작 불가

## 차원별 채점 항목

### 1. 요구사항 충족도 (필수) — 항목별 구현/부분/미구현 + file:line 근거
검색 화면: ①검색 결과 표시 ②빈 검색어 시 최근검색어 최대10 ③날짜 내림차순 ④삭제/전체삭제 ⑤재시작 후 유지(영속성) ⑥최근검색어 선택 시 검색
검색 결과 화면: ①List ②총 결과 수 ③저장소 정보(avatar/name/login) ④선택 시 WebView

### 2. 코드 품질·아키텍처
- TCA 단방향 흐름, State Equatable, Dependency 주입(raw 호출 금지)
- 모듈 경계 단방향 의존(App→Features→Domain/Core), Feature 간 직접 의존 없음
- 네이밍(Swift API Guidelines), 접근제어, 1파일1타입
- 에러 처리(도메인 에러 변환, ViewState), 효과 취소/디바운스
- 테스트 의미성(TestStore, 엣지 단언이 실제 동작 검증)

### 3. 완성도·동작
- 빌드 그린(`xcodebuild build`), 테스트 그린(전 스킴 0 failures), 플래키 없음
- 엣지케이스: 빈 결과/네트워크 에러/rate limit/페이지 경계
- AI 활용 흔적(정보용): 커밋 trailer, README AI 섹션, .Codex 환경 구성

### 4. UX·디자인 충실도 (코드↔png 개념 대조)
- 예시 png를 멀티모달로 읽고 우리 SwiftUI View의 레이아웃·요소 구성과 대조(픽셀 아님, 구조/요소 일치)
- 다크모드(시맨틱 컬러), 접근성(accessibilityLabel·Dynamic Type·44pt)

### 5. 추가 구현 (보너스)
- 자동완성 + 검색 날짜 노출(최근검색어 추출)
- scroll 중 Next Page prefetch + 로딩 상태

## 근거 요건 (중요)
- 모든 판정에 `file:line` 또는 png 영역 근거를 단다. 추측 금지 — 확인 못 하면 "확인 불가"로 명시하고 감점 사유에 기록.
- 빌드/테스트 점수는 실제 실행 결과(`xcodebuild`)로만. 코드만 보고 "통과 추정" 금지.
