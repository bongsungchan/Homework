# 과제: GitHub 저장소 검색

## 과제 개요

GitHub 저장소를 검색하는 앱을 구현하는 과제입니다.

- 문제 정의 능력, 아키텍처 설계, 데이터 구조, 알고리즘, 코드 품질, UX 등을 **종합적으로** 확인하는 것이 목표
- 컬리는 이 과제가 서로를 더 깊이 이해하기 위한 시작점이 되길 바람

### 평가 기준
- **개발 기초 역량**과 **AI 활용도**에 가장 큰 비중
- 추가 구현 항목은 평가에 반영되지만 비중은 크지 않음
- **기본 기능부터 순서대로** 구현하길 권장

---

## 요구사항 1. 검색 화면

### 기본 구현
1. 검색어 입력 후, 검색 결과를 보여준다.
2. 검색어가 비어있을 시, **최근 검색어를 최대 10개**까지 보여준다.
3. 최근 검색어는 **날짜 기준 내림차순** 정렬한다.
4. 최근 검색어 **‘삭제’ 또는 ‘전체 삭제’**가 가능하다.
5. 최근 검색 내역은 **앱 재시작 시에도 유지**된다. (영속성)
6. 최근 검색어 선택 시, 검색 결과를 보여준다.

### 추가 구현
1. 검색어 입력 시, **자동완성**을 보여준다.
2. 자동완성 노출 시, **검색 날짜**를 같이 보여준다.
3. 자동완성은 **최근 검색어에서 추출**하여 사용한다.

> 참고 이미지: `예시 1..png`
> - 검색어가 비어있을 때: "Search" 타이틀 + 검색 입력창 + "최근 검색" 목록(각 항목에 삭제 x 버튼) + "전체삭제" 버튼
> - 검색어 입력 시: 자동완성 항목(예: "Swift")과 우측에 검색 날짜(예: 03.09) 표시

## 요구사항 2. 검색 결과 화면

### 기본 구현
1. 검색 결과를 **List 형태**로 보여준다.
2. **총 검색 결과 수**를 보여준다.
3. 저장소 정보를 보여준다.
   - **Thumbnail**: `owner.avatar_url`
   - **Title**: `name`
   - **Description**: `owner.login`
4. 검색 결과 선택 시, **WebView**를 통해 해당 저장소로 이동한다.

### 추가 구현
1. Scroll 중간에 **Next Page를 미리 호출**한다. (페이지네이션 prefetch)
2. Next Page 로딩 시, **로딩 상태**를 보여준다.

### Endpoint
```
[GET] https://api.github.com/search/repositories?q={keyword}&page={page}

예시) https://api.github.com/search/repositories?q=swift&page=1
```

> 참고 이미지: `예시 2..png`
> - 상단에 검색창 + "266,714개 저장소" 형태의 총 결과 수
> - 리스트: 좌측 썸네일(아바타) + 저장소 name + owner.login

## 유의사항

- 본인을 가장 잘 표현할 수 있는 **Architecture**와 **UI 방식(SwiftUI, UIKit)**을 자유롭게 선택
- 개발 시 **AI Assist(ChatGPT, Gemini 등)를 충분히 활용**할 것
- 추후 인터뷰에서 **어떤 부분에 AI Assist를 활용했는지** 질의응답이 진행될 수 있음
- **‘검색 화면’ → ‘검색 결과 화면’ 순서**로 구현할 것
- 기한 내 완성하지 못하더라도 **완성한 부분만 제출**
- **GitHub을 사용해 이슈 관리 및 프로젝트를 제출**할 것
- 이해되지 않는 부분이 있을 경우 채용 담당자에게 문의
